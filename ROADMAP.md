# Roadmap

Из конкурентного разбора (Rose, Combot, CAS, Shieldy, KolasAI, SmartMod) — см. обсуждение в чате.
Порядок = приоритет.

## 1. Капча для новых участников — `готово`
Отраслевой стандарт (есть у Rose, Shieldy, SmartMod), у нас не было вообще.
**Выключена по умолчанию**, в Mini App убрана в свёрнутый раздел «Дополнительно»
(не на первом экране настроек, чтобы не перегружать базовый флоу).

- `captchaEnabled: boolean` (default `false`) в настройках группы
- При включении: новый участник мутится + получает кнопку-подтверждение
  «Я не бот» в группе (2 минуты). Не нажал вовремя — кик при следующем
  сообщении в группе (лениво, без отдельного воркера/крона — serverless-деплой
  без персистентного процесса, см. `lib/telegram/captcha.ts`)
- Чат-команда `/captcha on|off`
- Mini App: тумблер в свёрнутой секции «Дополнительно» (по умолчанию закрыта)
- Осведомлённость о правах бота (`missingPermissionsFor`) теперь учитывает
  и капчу — она тоже требует право «Блокировка участников»

## 2. Приветственные сообщения — `готово`
- `welcomeEnabled: boolean`, `welcomeMessage: string | null`
- Плейсхолдер `{user}` → HTML-упоминание нового участника
- Команда `/welcome <текст>` / `/welcome off`
- Поле ввода в Mini App (видимо на основном экране настроек, не спрятано)

## 3. Расширенная статистика (не только нарушения) — `готово`
- Новые счётчики за день: `messages`, `joins` (`lib/db/stats.ts` → `incrementActivity`/`getActivity`)
- Отдельная секция «Активность» на экране статистики Mini App, над блоком нарушений

## 4. Явное позиционирование uz-cyrl + ru — `сделано частично`
В ходе разбора рынка не нашёл ни одного конкурента, целящегося именно в
связку узбекский-кириллица + русский. Открытая ниша.

- Описание бота (`setMyDescription`) можно усилить акцентом на это
- README уже описывает локализацию как основную фичу, а не довесок

## 5. Платный тариф на Groq-квоту — `план, не реализовано`
Сейчас премиум-режим бесплатен, потому что укладываемся в free tier Groq.
При росте числа премиум-групп это станет реальным узким местом (уже есть
RPM/TPM/RPD-бюджеты в `lib/moderation/groq.ts` — они же станут границей тарифа).

**Не начинаю кодить без вашего решения по:**
- провайдер платежей (Telegram Stars нативно поддерживается ботами — проще
  всего; либо внешний Stripe/ЮKassa)
- ценовые пороги (сколько премиум-проверок в месяц входит в бесплатный тир)
- нужна ли отдельная "команда/тариф", или это функция на уровне группы

Как только определитесь — реализация: новое поле `premiumQuotaTier` в
`GroupSettings`, отдельный Redis-бюджет на тариф вместо общего пула,
интеграция `sendInvoice`/Stars в бота.
